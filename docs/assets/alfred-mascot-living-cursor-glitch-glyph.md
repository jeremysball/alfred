## Living Cursor

### Main Illustration

```
Create a software mascot illustration: a living text cursor for a memory assistant called Alfred.

FORM: A vertical text cursor bar — thick, substantial, not a thin line. Proportions like a terminal cursor: taller than wide, about 4:1 ratio. Cyan color (#00d9ff) with subtle inner glow.

THE FACE: Near the top third of the cursor, two small dark circular eyes and a simple curved line mouth. Expressions:
- Neutral: flat mouth, calm eyes
- Alert: slightly raised eyebrows (small lines above eyes), attentive
- Pleased: small upward curve mouth

ANIMATION SUGGESTION: Show the cursor "breathing" — subtle pulse glow. At the base, small cyan particles trail downward like falling pixels, representing memories being processed.

STYLE: Clean vector illustration with terminal aesthetic. Belongs in a dark IDE or terminal window. Smooth but with slight pixel-grid awareness — feels digital-native.

COLORS:
- Cursor body: cyan (#00d9ff)
- Glow: lighter cyan/white center
- Eyes/mouth: dark navy (#0a1628) or black
- Particles: cyan fading to transparent
- Background: dark navy (#0a1628) or transparent

VIBE: The cursor that's been with you through every line of code. Quiet companion. Remembers your typos and triumphs.

TECHNICAL: Full illustration, 1024x1024px. Character centered, vertical orientation.
```

## Glitch Glyph

### Main Illustration

```
Create a software mascot illustration: a glitch glyph for a memory assistant called Alfred.

FORM: An abstract character that looks like a corrupted Unicode symbol or font rendering error. Suggests a stylized letter "A" or abstract face, but wrong — slightly offset, doubled edges, not quite stable.

THE GLITCH: Single-pixel offsets, color channel splits (cyan and amber shadows), scan lines, minor static. Not broken — just not from this character set. Edges vibrate subtly.

FACE: The glyph IS the face. The "A" shape forms brows and nose. Negative space creates eyes. Crossbar is mouth. Readable as letter AND face simultaneously.

STRUCTURE: Central vertical spine, two diagonal arms (like A), but asymmetrical. One arm longer. One side has "damage" — missing pixels, offset duplicate.

STYLE: Bitmap aesthetic executed in vector. Retro computing, terminal art, demoscene. Feels like a bug that became a feature.

COLORS:
- Main glyph: cyan (#00d9ff)
- Glitch shadow 1: amber (#ffb84d), offset 1px right
- Glitch shadow 2: dark cyan, offset 1px left
- Scan lines: subtle dark overlay
- Background: dark navy (#0a1628)

VIBE: A character that lives in the space between pixels. Remembers what the text forgets. Digital native, unexpected.

TECHNICAL: Centered composition. 1024x1024px. The glitch should feel intentional, not random noise.
```

### State Variations

```
Create 4 state variations of the Glitch Glyph mascot arranged in 2x2 grid.

BASE: Same abstract glitchy A-shape glyph.

STATES:
1. STABLE: Minimal glitch, holds form, calm expression, single slight offset shadow
2. PROCESSING: More artifacting, edges vibrating, multiple offset shadows, brighter glow
3. RECALLING: Fragmented momentarily — pieces spread apart then rejoin, chaotic energy
4. RESOLVED: Symmetric, calm, all glitches aligned meaningfully into pattern

STYLE: Consistent glitch aesthetic. Each state shows different "stability" of the form.

LAYOUT: 2x2 grid, equal size for each state.

COLORS: Cyan base, amber glitch shadows, dark navy background. States vary in intensity.
```

### Glyph Detail Close-up

```
Create a detailed close-up of the Glitch Glyph mascot showing the "face" formed by negative space.

STYLE: Extreme close-up, technical illustration. Shows the intentional design of "face in abstract shape."

COLORS: Cyan, amber, dark navy. High contrast for detail visibility.
```
